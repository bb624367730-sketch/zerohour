## Community Strategy

### Phase 1: Pre-Launch (Current)

**Goals**: Build awareness, gather waitlist, find early adopters.

**Actions**:
1. **Twitter/X account** — Post game previews, screen recordings, countdown teasers. Follow and engage with Sui ecosystem accounts.
2. **Sui ecosystem directories** — List on Sui dApp store, Sui Wiki, DappRadar.
3. **Waitlist** — Simple Google Form or Discord bot to collect wallet addresses for launch notification.
4. **Content**: Screenshots of the UI, short gameplay videos, team comparison graphics.

**Timeline**: 1-2 weeks before mainnet.

### Phase 2: Mainnet Launch

**Goals**: Drive initial volume, establish game as active dApp.

**Actions**:
1. **Launch announcement** — Coordinated post across Twitter, Discord, Telegram, Reddit (r/sui).
2. **First-round seeding** — Seed the first round with community fund to bootstrap activity.
3. **Launch event** — "First 24 Hours" event: track stats publicly, create FOMO with live countdown.
4. **Referral push** — Encourage sharing referral links — 10% of ticket price goes to referrer.
5. **KOL outreach** — DM 5-10 mid-size Sui/KOL accounts, offer to demo the game. Authentic content > paid shills.

**Timeline**: Launch week.

### Phase 3: Growth

**Goals**: Sustained player base, recurring volume.

**Actions**:
1. **Weekly round recaps** — Tweet stats: total volume, winner, biggest team, most tickets bought. Creates narrative.
2. **Meme contests** — Best meme about a team wins SUI. Drives engagement, UGC.
3. **Team rivalries** — Lean into the Doge vs Pepe vs Monkey vs Cat tribalism. Polls, leaderboards, trash talk.
4. **Feature updates** — Announce improvements publicly. Show the project is alive.
5. **Airdrop events** — Special rounds with boosted airdrop pool. Announce in advance to drive volume.

**Timeline**: Ongoing after launch.

### Phase 4: Maturity

**Goals**: Self-sustaining community, ecosystem integration.

**Actions**:
1. **Governance discussion** — Community votes on parameter changes (fee adjustments, new teams).
2. **Cross-promotion** — Partner with other Sui dApps for cross-marketing.
3. **Analytics dashboard** — Public Dune/flipside dashboard for game stats.

## FAQ

**Q: Is this gambling?**  
A: ZeroHour is a strategy game. Team choice, round timing, and meta analysis all affect outcomes. No house edge — the contract takes no profit beyond the transparent 2% community fund.

**Q: How is this different from the original Fomo3D?**  
A: Built on Sui instead of Ethereum — meaning no gas wars, no block stuffing, true on-chain randomness, and sub-cent transaction fees.

**Q: Can the devs steal the jackpot?**  
A: No. The contract has no admin withdrawal function for player funds. All allocations are hardcoded and verifiable on-chain.

**Q: What happens if nobody buys for 24 hours?**  
A: The countdown expires and the last buyer wins. The timer maxes out at 24 hours to guarantee the game eventually ends.

**Q: Is there a token?**  
A: No. ZeroHour uses native SUI only. No token dilution, no ponzi tokenomics. Real yield from game volume.
